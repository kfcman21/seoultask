업무문서, 인터랙티브 웹페이지로 변신시키기

NotebookLM과 Gemini를 활용하여 딱딱한 업무 문서를 교사의 눈높이에 맞는 생동감 있는 교육 자료로 만드는 방법을 단계별로 알아봅니다.

🔢 AI를 활용한 웹페이지 제작 단계

📂 1-2단계: 자료 탑재 및 시각화

NotebookLM에 원본 PDF 파일을 업로드하고, '마인드맵' 기능을 이용해 핵심 주제를 추출합니다.

💬 3단계: 프롬프트 입력

추출된 주제를 바탕으로, 교사 연수 자료 형태의 슬라이드 제작을 요청하는 구체적인 프롬프트를 입력합니다.

📋 4단계: 결과물 복사

NotebookLM이 생성해준 슬라이드 형식의 텍스트 결과를 전체 복사합니다.

🤖 5단계: 웹페이지 변환

'웹페이지 전문가' Gemini에게 복사한 텍스트를 제공하며 인터랙티브 웹페이지 코드 생성을 요청합니다.

💾 6단계: HTML 파일로 저장

생성된 HTML 코드를 메모장에 붙여넣고, 정책1.html과 같이 `.html` 확장자로 저장합니다.

🔁 7단계: 주제별 반복

다른 핵심 주제(미래역량교육, 교육공동체 등)에 대해서도 3~6단계 과정을 반복하여 각각의 HTML 파일을 만듭니다.

핵심 포인트!

이 과정의 핵심은 AI 도구를 단계적으로 연계하여, 아이디어 구체화부터 최종 결과물 제작까지 자동화하는 것입니다. 각 AI의 장점을 최대한 활용하는 것이 중요합니다.

🌐 8단계: GitHub 무료 웹 호스팅

GitHub Pages 기능을 이용하면 누구나 무료로 자신의 HTML 파일을 웹에 게시할 수 있습니다. 아래 절차를 따라 해보세요.

  • 회원가입 및 로그인: GitHub 사이트에 접속하여 계정을 생성합니다.
  • 저장소(Repository) 생성: 'New repository'를 클릭하여 새로운 저장소를 만듭니다. (이름은 자유, Public으로 설정)
  • 파일 업로드 및 이름 변경:
    1. 생성된 저장소에서 'Add file' > 'Upload files'를 선택하여 만들어둔 HTML 파일을 올립니다.
    2. 업로드된 파일의 이름을 index.html로 변경합니다. 이 이름이 웹사이트의 첫 페이지가 됩니다.
  • 페이지 활성화: 저장소 상단의 'Settings' 탭 > 좌측 메뉴의 'Pages'로 이동합니다.
  • 소스(Source) 설정: 'Branch'를 'None'에서 'main'으로 변경하고 'Save' 버튼을 누릅니다.
  • 주소 확인: 약 1분 정도 기다리면, 페이지 상단에 초록색 배경으로 게시된 사이트 주소가 나타납니다.

✨ 9단계: 최종 결과물 취합 및 공유

위 과정을 통해 생성된 3개의 웹페이지 주소를 Canva와 같은 디자인 툴을 이용해 하나의 페이지로 묶고, QR 코드를 생성하여 편리하게 공유할 수 있습니다.
